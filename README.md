Recipe Cuisine Classification

Реализация системы машинного обучения для автоматического определения национальной кухни блюда на основе списка его ингредиентов.

1. Данные (Data Source)

В проекте используется открытый набор данных \[What's Cooking?](https://www.kaggle.com/datasets/kaggle/recipe-ingredients-dataset) с платформы Kaggle.

Файл: `train.json` (39,774 рецепта).

Формат: JSON, содержащий ID рецепта, целевую категорию (`cuisine`) и список ингредиентов (`ingredients`).

Инструкция: Для запуска кода скачайте файл `train.json` и поместите его в корень проекта (или в папку `data/`).

2. Реализация и выбор классификатора

В ходе работы было реализовано и сравнено 5 различных классификаторов. Для каждого была рассчитана метрика Accuracy (точность).

Результаты первичного сравнения:

Классификатор				Точность (Accuracy)

Linear SVM (Линейное ядро)		0.7819

LightGBM (Градиентный бустинг)		0.7555

Extra Trees Classifier			0.7378

K-Nearest Neighbors			0.6255

Dummy Classifier (Базовая линия)	0.1970

Победитель: Linear SVM.** Данный алгоритм показал лучший результат, так как он эффективно работает с разреженными текстовыми данными (TF-IDF матрицами), где количество признаков (ингредиентов) достигает нескольких тысяч.

3. Повышение точности (Optimization)

Для улучшения базового результата (0.7819) были предприняты следующие шаги:

Лемматизация (NLP): Использование библиотеки nltk для приведения слов к начальной форме (например, tomatoes -> tomato).

Настройка гиперпараметров: С помощью GridSearchCV был проведен поиск оптимального параметра регуляризации C.

Ансамблирование: Создание Voting Classifier (объединение SVM, Logistic Regression и SGD).

Итоговая точность после оптимизации: 0.7874


4. Тестирование и визуализация

Матрица ошибок: Построена Confusion Matrix, позволившая выявить наиболее похожие кухни (например, путаница между Thai и Vietnamese).

Custom Test: Проведена проверка на собственных данных:

Ввод: tortilla, beans, meat, salsa

Предсказание: MEXICAN (Верно).
